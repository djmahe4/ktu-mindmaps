# ⚡ Fast Conditions Cheat Sheet

## 🔹 1. Single Loop

### ✅ Linear → **O(n)**

👉 Condition:

* `i++`, `i--`, `i += k`, `i -= k`

💡 Rule:

> Moves one step each time

**Example**

```c
for(i=1; i<=n; i++)
```

---

### ✅ Logarithmic → **O(log n)**

👉 Condition:

* `i *= c`, `i /= c` (c > 1)

💡 Rule:

> Value grows/shrinks exponentially

**Example**

```c
for(i=1; i<=n; i*=2)
```

---

### ✅ Square Root → **O(√n)**

👉 Condition:

* `i * i <= n` or `i <= √n`

💡 Rule:

> Loop runs until the square of the index reaches n

**Example**

```c
for(i=1; i*i<=n; i++)
```

---

## 🔹 2. Nested Loops

### ✅ Independent → **O(n²)**

👉 Condition:

* Inner loop does **not depend on outer**

💡 Rule:

> Full loop runs every time

**Example**

```c
for(i=1; i<=n; i++)
  for(j=1; j<=n; j++)
```

---

### ✅ Dependent → **O(n²)** (still)

👉 Condition:

* Inner loop depends on `i` (like `j <= i`)

💡 Rule:

> Forms triangle → sum = n²/2

**Example**

```c
for(i=1; i<=n; i++)
  for(j=1; j<=i; j++)
```

---

## 🔹 3. Mixed Pattern

### ✅ Linear + Log → **O(n log n)**

👉 Condition:

* One loop linear, inner loop logarithmic

**Example**

```c
for(i=1; i<=n; i++)
  for(j=1; j<=n; j*=2)
```

---

## 🔹 4. Quick Recognition Rules

* ➕ Add loops → **O(n + n) = O(n)**
* ✖️ Nested loops → **multiply → O(n × n) = O(n²)**
* 🔁 Halving/doubling → **log n**
* 🔺 Triangle loops → **O(n²)**
* 🔢 Index squared check → **√n**

### Properties of Asymptotic Notations

Asymptotic notations possess several properties that are useful in algorithm analysis:

*   **Reflexivity:** A function is in the same asymptotic class as itself.
    *   `f(n) = O(f(n))`
    *   `f(n) = Ω(f(n))`
    *   `f(n) = Θ(f(n))`
*   **Symmetry:** If `f(n)` is tightly bound by `g(n)`, then `g(n)` is also tightly bound by `f(n)`.
    *   `f(n) = Θ(g(n))` if and only if `g(n) = Θ(f(n))`
*   **Transpose Symmetry:** Relates upper and lower bounds.
    *   `f(n) = O(g(n))` if and only if `g(n) = Ω(f(n))`
    *   `f(n) = o(g(n))` if and only if `g(n) = ω(f(n))`
*   **Transitivity:** If `f(n)` is bounded by `g(n)`, and `g(n)` is bounded by `h(n)`, then `f(n)` is also bounded by `h(n)`.
    *   If `f(n) = O(g(n))` and `g(n) = O(h(n))`, then `f(n) = O(h(n))`
    *   If `f(n) = Ω(g(n))` and `g(n) = Ω(h(n))`, then `f(n) = Ω(h(n))`
    *   If `f(n) = Θ(g(n))` and `g(n) = Θ(h(n))`, then `f(n) = Θ(h(n))`
    *   If `f(n) = o(g(n))` and `g(n) = o(h(n))`, then `f(n) = o(h(n))`
    *   If `f(n) = ω(g(n))` and `g(n) = ω(h(n))`, then `f(n) = ω(h(n))`

These notations are crucial for comparing algorithms and predicting their performance for large inputs, abstracting away machine-dependent factors and implementation details.
